# Notion 第二大脑 - 设置指南

本指南将帮助你从零开始搭建 Notion 第二大脑系统。

## 前置准备

### 1. 创建 Notion Integration

1. 访问 https://www.notion.so/my-integrations
2. 点击 "+ New integration"
3. 填写基本信息：
   - Name: "Second Brain"
   - Associated workspace: 选择你的工作区
   - Type: Internal
4. 点击 "Submit" 创建
5. 复制 "Internal Integration Token"（以 `secret_` 开头）

### 2. 准备根页面

1. 在 Notion 中创建一个新页面，命名为 "Second Brain" 或其他你喜欢的名字
2. 点击页面右上角的 "..." 菜单
3. 选择 "Add connections"
4. 搜索并添加刚才创建的 Integration

### 3. 获取页面 ID

1. 在浏览器中打开刚才创建的页面
2. 查看 URL，格式为：`https://www.notion.so/Your-Page-Name-{PAGE_ID}`
3. 复制 `PAGE_ID` 部分（32位字符，可能包含连字符）

## 安装步骤

### 步骤 1: 克隆项目

```bash
git clone https://github.com/yourusername/notion-second-brain.git
cd notion-second-brain
```

### 步骤 2: 安装依赖

```bash
npm install
```

### 步骤 3: 配置环境变量

```bash
cp .env.example .env
```

编辑 `.env` 文件，填入你的信息：

```env
NOTION_API_TOKEN=secret_your_token_here
TIMEZONE=Asia/Shanghai
DAILY_NOTE_TIME=09:00
AUTO_ARCHIVE_DAYS=30
```

### 步骤 4: 运行初始化

```bash
npm run setup
```

按照提示操作：
1. 输入根页面 ID
2. 确认创建结构
3. 等待初始化完成

初始化完成后，会在项目根目录生成 `.notion-config.json` 文件，包含所有创建的页面和数据库 ID。

## PARA 结构说明

### Projects（项目）

**定义**：有明确截止日期和可交付成果的短期目标

**示例**：
- 完成季度报告
- 学习 React 框架
- 筹备生日派对

**使用建议**：
- 项目应该是可完成的，有明确的结束点
- 定期回顾项目状态
- 完成后归档，不要让项目列表过长

### Areas（领域）

**定义**：需要持续维护的长期责任和兴趣

**示例**：
- 健康管理
- 财务规划
- 职业发展
- 人际关系

**使用建议**：
- 领域是持续的，没有结束日期
- 为每个领域设定标准（什么状态是"好的"）
- 定期审视是否需要新增或移除领域

### Resources（资源）

**定义**：感兴趣的主题和未来可能用到的参考资料

**示例**：
- 编程教程
- 设计灵感
- 烹饪食谱
- 投资理财知识

**使用建议**：
- 资源是被动的，不需要行动
- 添加标签方便日后检索
- 定期清理不再相关的资源

### Archives（归档）

**定义**：已完成或不再活跃的内容

**使用建议**：
- 定期归档已完成的项目
- 保留归档以便未来参考
- 不要删除，归档即可

## 每日工作流

### 早晨（5-10分钟）

1. 创建今日笔记
   ```bash
   npm run daily-note
   ```

2. 在 Notion 中打开今日笔记

3. 查看仪表盘，了解：
   - 待处理的 Inbox 项目
   - 今日待办事项
   - 当前进行的项目

4. 规划今日重点

### 白天（实时）

1. **捕捉想法**
   - 使用 Notion Mobile App 快速记录
   - 所有内容先进 Inbox
   - 不要停下来分类，保持思维流畅

2. **处理信息**
   - 使用 Web Clipper 保存网页
   - 邮件转发到 Notion
   - 添加书签到 Resources

### 晚上（10-15分钟）

1. **整理 Inbox**
   - 处理今天收集的内容
   - 分类到 PARA 体系
   - 添加标签和链接

2. **更新项目**
   - 更新项目进度
   - 添加今日完成的任务
   - 记录遇到的问题和想法

3. **每日回顾**
   - 完成今日笔记的总结部分
   - 记录今日亮点
   - 计划明日重点

## 每周工作流

### 周日晚上（30-60分钟）

1. **回顾本周**
   - 查看本周的每日笔记
   - 总结本周成果
   - 识别模式和趋势

2. **整理项目**
   - 更新项目状态
   - 归档已完成项目
   - 调整优先级

3. **清理 Inbox**
   - 确保 Inbox 清空
   - 重新分类内容
   - 删除无用信息

4. **规划下周**
   - 设置下周目标
   - 安排重要任务
   - 准备需要的资源

## 每月工作流

### 月初（1-2小时）

1. **月度回顾**
   - 回顾上月项目完成情况
   - 检查各领域的状态
   - 分析时间分配

2. **目标调整**
   - 根据季度目标调整月度重点
   - 更新项目列表
   - 移除不再相关的内容

3. **系统维护**
   - 清理 Archives
   - 优化标签系统
   - 更新模板

## 高级技巧

### 1. 建立双向链接

在笔记中提及其他页面时，使用 `@` 提及：
- 这会自动创建链接
- 被链接的页面会显示反向链接
- 帮助建立知识网络

### 2. 使用数据库视图

为不同场景创建不同视图：
- **今日待办**：筛选出今天要做的任务
- **本周项目**：显示本周活跃的项目
- **高优先级**：只显示重要紧急的内容

### 3. 标签策略

建立一致的标签体系：
- **状态标签**：进行中、已完成、暂停
- **类型标签**：学习、工作、个人
- **主题标签**：技术、设计、管理
- **优先级标签**：高、中、低

### 4. 模板使用

为常见内容类型创建模板：
- 会议记录
- 读书笔记
- 项目计划
- 决策日志

### 5. 自动化

设置定时任务：

```bash
# Linux/Mac - 使用 cron
crontab -e

# 每天早上9点创建每日笔记
0 9 * * * cd /path/to/notion-second-brain && npm run daily-note

# 每周日晚上执行自动归档
0 20 * * 0 cd /path/to/notion-second-brain && npm run auto-archive
```

## 常见问题

### Q: 如何备份数据？

A: 运行备份脚本：
```bash
npm run backup
```

### Q: 如何迁移已有笔记？

A: 参考 `docs/migration-guide.md`

### Q: API 速率限制怎么办？

A: Notion API 限制每秒 3 个请求，脚本已内置延迟处理。如果仍遇到限制，可以增加延迟时间。

### Q: 如何自定义模板？

A: 编辑 `config/templates.json` 文件，按照 Notion Block 格式添加新模板。

### Q: 忘记整理怎么办？

A: 不要给自己太大压力！第二大脑的核心是捕捉，整理可以批量进行。设置每周固定时间处理即可。

## 最佳实践

1. **保持简单**：从基础结构开始，逐步优化
2. **定期回顾**：建立固定的回顾习惯
3. **及时捕捉**：不要让想法溜走
4. **建立连接**：多使用双向链接
5. **持续迭代**：系统应该随着你的需求进化
